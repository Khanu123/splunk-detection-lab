# Employer Review Guide - Splunk Detection Lab

## What this project proves

This project is designed to show readiness for **SOC Analyst / SIEM Analyst** work. It demonstrates practical defensive security judgement, clear documentation, and the ability to turn technical findings into evidence an analyst or manager can review.

## Skills demonstrated

Splunk SPL, authentication log analysis, brute-force detection, password-spray detection, analyst playbooks

## Best evidence to review

SPL detections, synthetic auth logs, screenshot preview, Python validation, unit tests, analyst playbook

## Why it is relevant

Shows practical SIEM thinking: what data is needed, how to detect abuse, and how an analyst should investigate.

## How I would discuss this in an interview

- What data the tool or lab needs before it can produce useful results.
- Which findings should be prioritised first and why.
- Where false positives can happen and how I would validate them.
- How the output would help a SOC analyst, cloud security analyst, or security operations team make a decision.
- What I would improve if this became a production team tool.

## Professional boundaries

This repository uses safe sample data and defensive logic. It is intended for portfolio review, learning, and interview discussion. It does not include exploit instructions, credential misuse, or guidance for unauthorized activity.
